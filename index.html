CAPÍTULO 2 - COMO A INTERNET FUNCIONA

Sinal, Roteador, Cliente e Servidor

  • A internet é uma rede gigantesca de ligação entre outras redes ao redor do mundo
  • A maioria das transmissões entre os continentes é realizada por cabos submarinos
  • Submarine Cable Map
  • Já do litoral para dentro do continente a transmissão é feita por satélites e balões
  • No continente a sistema se liga a outros sistemas de comunicação como o de telefonia
  • Esses sistemas transmitem sinais diferentes
  • É realizado a conversão do formato das ondas de telefnia por exemplo para as ondas aceitas pelo computador (bites)
  • Essa conversão é feita pelo modem que modula e demodula o sinal
  • Roteadores definem a melhor rota para os dados, evitando congestionamentos
  • Cliente conecta no servidor
  • Porém, como o a internet descobre qual servidor do mundo esta a informação?
  • O protocolo TCP/IP garante que todas as comunicações seguem o mesmo padrão
  • permite dispositivos diferentes, com tecnologias diferentes trocarem informações
  • O TCP/IP conecta os nós, que nada mais são do que os pontos conectados a rede
  • Ao se conectar a internet você recebe um endereço de IP
  • O DNS, Domain Name System é como uma lista dinâmica gigantesca que liga os IPS aos nomes dos sites
  • Cliente tenta acessar um servidor
  • Internet verifica no DNS qual o IP atrelado ao nome do site digitado
  • O navegador pode então acessar o endereço correto
  • A solicitação chega ao servidor e ele envia uma copia do documento para o cliente

Domínio e Hospedagem

  • A parte principal de uma URL (Uniform Resource Locator) é o domínio.
  • Domínio: curitiba.pr.gov.br
  • O .br é o TLD, Top Level Domain
  • GTLD são TLDs genéricos, sem indicação de país, exemplos: .com, .gov, .net, etc.
  • ccTLD (Country Code) são as TLDs com designação de países, exemplos: .br, .us, etc.
  • O www é um sub-domínio e podem haver outros como images.google.com ou mail.google.com
  • O www não faz parte do domínio
  • Uma URL também tem o protocolo (http ou https)
  • O https possui a segurança por SSL (serviço de criptografia de dados)

Hospedagem e Data Centers

Os data centers são a infraestrutura necessária para manter servidores sempre ligados e disponíveis e forncecer espação para o armazenamento das informações

CAPÍTULO 3 - COMO FUNCIONA A HTML E AS CSS

Tags HTML

  • Não se programa em HTML
  • HTML não funciona com instruções e sim com marcações em tags
  • HTML = Conteúdo
  • CSS = Estilo
  • JAVASCRIPT = Interação
  • Em <a href="destino.html">Exemplo</a>
  • href é um parâmetro
  • destino.html é um valor do parâmetro
  • Exemplo é o conteúdo

Sletores CSS


	P {
		color: blue;
	}
  • p é o seletor
  • color:blue é a declaração
  • color é a propriedade
  • blue é o valor

    Formato de imagens

    • JPEG - tamanho reduzido, ocupam pouco espaço
    • PNG - capacidade de opacidade, deixando fund transparente

    Sempre usar .jpeg para as imagens, com compactação entra 30% a 50% para economizar espaço e processamento. PNG só deve ser utilizado quando houver a necessidade do fundo transparente

    Tamanho / resolução

    A regra é: Sempre use as imagens com o tamanho necessário. Se precisa ser 200x200 use uma imagem com 200x200, isso economiza espaço e o google valoriza sites rápidos e leves em sua busca

    Texto alternativo da imagem (Parâmetro alt="")

    Auxiliam na indexação do site no google e na acessibilidade. O ideal é ser objetivo. O texto também auxilia o google a saber o que tem na sua foto e exibi-la no google imagens

    Ícones Favicon

    Sempre divida os textos em títulos e subtítulos, isso torna a leitura menos cansativa. Os títulos no HTML (h1, h2, h3, h4, h5 e h6) devem ser usados justamente como uma hierarquia, onde h6 pertence a h5, h5 pertence a h4 e assim por diante, para favorecer a semântica.

    CAPÍTULO 8 - FORMATAÇÃO DE TEXTOS

    Semântica é o siginificado dos vocábulos por oposição à sua forma.

    Siginificado de algumas tags semânticas:

    • <strong> : O termo possui força dentro da frase
    • <em> : O termo deve ser enfatizado
    • <mark> : O termo deve ser mais enfatizado ainda
    • <small> : Normalmente utilizado para as letras pequenas, como em contratos
    • <del> : Siginifica texto que pode ser lido, porém pode ser também deixado para lá desconsiderado
    • <ins> : O oposto do <del>, trata-se de um texto que deve ser lido e prestado atenção
    • <sup> : X5
    • <sub> : H2O
    • <code> : Indicam ao navegador que se trata de um código
    • <pre> : Tag que ajuda na formatação do <code> mantendo o texto pré formatado, indicando por exemplo a identação
    • <q> : Para indicar citações geralmente no meio de parágrafos. São trechos de textos escritos ou ditos por outra pessoa
    • &lblockquote> :
      Para citações mais longas que normalmente possuem um parágrafo apenas para si.
    • <abbr title="Hyper Text Markup Language"e>HTML</abbr> : Para deixar claro ao navegador e ao usuário siginificado de uma sigla: HTML

    É também possível juntar tags dentro de outras, potêncializando os efeitos. Exemplo: <strong><emg></em></strong> geram um texto com negrito (força) e itálico (ênfase).

    CAPÍTULO 10 - LIGAÇÕES EM TODA PARTE

    Os mecanismos de busca utilizam os hyperlinks de sites para encontrar outros sites

    • Atributo hreflang - Define a linguagem do site do link para auxiliar possíveis traduções
    • Atributo target - Para definir se a página va abrir em nova aba ou não
    • Atributo rel
    • Valor "next" - Indica que o link é para a próxima parte do documento atual
    • Valor "prev" - Indica que o link é para a parte anterior do documento atual
    • Valor "author" - Indica que o link é para o site do autor do artigo atual"
    • Valor "external" - Indica que o link é para outro site que não faz parte do site atual
    • Valor "nofollow" - Indica que o link é para outro site não endossado, como um link pago
    • Atributo dowload - Difine o download do arquivo definido
